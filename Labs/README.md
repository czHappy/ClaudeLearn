## 给出题者的使用指南

### 测试流程

1. **准备阶段**（5分钟前）
   - 将 `循环编程题目.md` 发送给学生
   - 确保学生的电脑已安装 C++ 编译器（g++ 或 Visual Studio）
   - 告知学生创建 `answer.cpp` 文件来写代码

2. **阅读材料**（10分钟）
   - 让学生认真阅读"第一部分：学习材料"
   - 可以提问，但不直接给出答案

3. **独立编程**（20-30分钟）
   - 学生独立完成"第二部分：编程题目"
   - 如果卡住超过 10 分钟，可以引导查看提示
   - 提示按顺序开放：提示1 → 提示2 → 提示3

4. **测试验收**（5分钟）
   - 让学生自己编译运行，测试所有示例
   - 检查输出格式是否完全一致

## 如何评分

### 快速评分法（推荐）

使用以下命令批量测试学生代码：

```bash
# 编译学生代码
g++ student_answer.cpp -o student_answer

# 测试用例1: n=3
echo "3" | ./student_answer > output1.txt
diff output1.txt expected1.txt  # 需要提前准备期望输出

# 测试用例2: n=5
echo "5" | ./student_answer > output2.txt

# 测试用例3: n=1
echo "1" | ./student_answer > output3.txt
```

## 常见问题处理

### Q1: 学生说"编译失败"？
**排查步骤**：
1. 检查是否漏了分号 `;`
2. 检查花括号 `{}` 是否配对
3. 检查是否写了 `using namespace std;`
4. 检查变量名是否拼写正确

### Q2: 输出格式不对？
**常见错误**：
- 忘记在数字之间加空格
- 忘记在每行结束时换行（`endl`）
- 多输出或少输出了数字

### Q3: 学生完全不会循环？
**引导方式**：
1. 先让他用循环打印 5 个星号 `*****`
2. 再让他用循环打印 1 到 5
3. 然后引导他思考"如何打印两行"
4. 最后引导他理解嵌套循环

### Q4: 如何判断是否"现学现用"？
**观察要点**：
- 查看学习材料的时间（应该认真阅读 5-10 分钟）
- 编程时是否频繁回看材料（正常现象）
- 是否独立思考，还是完全依赖提示
- 错误后是否能自己调试

## 题目设计说明

### 为什么选择这道题？

1. **循环核心知识**
   - 基础 for 循环
   - 嵌套循环（两层）
   - 循环变量的使用

2. **思维挑战度适中**
   - 不是简单的重复打印
   - 需要观察规律（第 i 行打印 i 个数）
   - 需要理解内外层循环的关系

3. **现学现用能力**
   - 学习材料涵盖所有必需知识
   - 有示例代码可以参考
   - 需要学生自己组合和应用

4. **易于评分**
   - 输出结果明确
   - 可以自动化测试
   - 没有多种"正确答案"的歧义

### 难度梯度

- **基础任务**：适合 70% 的零基础学生在 30 分钟内完成
- **拓展挑战**：适合 20% 的快速学习者，展示更强的理解力
- **提示系统**：帮助卡住的学生，确保 90% 的人能完成

## 建议的改进方向

如果觉得题目太难或太简单，可以调整：

### 降低难度：
- 去掉数字之间的空格要求（直接 `cout << j;`）
- 减少测试用例数量
- 提供更完整的代码框架

### 提高难度：
- 要求输出九九乘法表
- 要求输出菱形图案
- 限制不能使用某些语句（如只能用 while 不能用 for）
