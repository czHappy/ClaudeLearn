### 第2题（10分）

**阅读材料**

1945年，数学家冯·诺依曼提出了一个革命性的想法：为什么不能让计算机把程序和数据存储在同一个地方呢？

在此之前，计算机的程序是通过重新连接电路或打孔纸带来设置的，每次改变程序都要重新布线，费时费力。而冯·诺依曼提出，可以把程序指令也当作"数据"存储在存储器中，计算机只需要按顺序读取这些指令并执行就可以了。

这个想法看似简单，却带来了巨大的变化。就像我们今天用手机时，只需要点击不同的App图标，手机就能切换功能——这正是因为每个App的程序都存储在手机的存储器里，需要时就调出来运行。

冯·诺依曼还提出，计算机应该包含五个基本部分：**运算器**（做加减乘除等计算）、**控制器**（指挥各部分协调工作）、**存储器**（存放程序和数据）、**输入设备**（如键盘鼠标）、**输出设备**（如显示器）。这个结构一直沿用至今，我们称之为"冯·诺依曼体系结构"。

**问题**：根据文章内容，下列理解最准确的是？

A. 冯·诺依曼之前的计算机无法存储数据，只能即时计算
B. 冯·诺依曼体系结构的核心创新是将程序和数据都存储在存储器中
C. 今天的智能手机不再使用冯·诺依曼体系结构，而是更先进的架构
D. 运算器是冯·诺依曼体系结构中最重要的部分，其他部分都是辅助性的

### 第3题（10分）

**阅读材料**

计算机有两个重要的存储空间：内存和硬盘(磁盘)。它们就像我们学习时用的课桌和书柜。

内存就像课桌的桌面——面积有限,但东西放在上面随手就能拿到,非常方便。你正在做的作业、翻开的课本都放在桌面上,需要什么立刻就能找到。但是下课了(关机了),桌面就要收拾干净,东西都得收起来。

硬盘就像书柜——空间很大,可以存放很多书本和资料,就算放一年也不会丢失。但每次要用的时候,都得从书柜里翻找出来,再搬到桌面上才能使用,这个过程比较慢。

所以,计算机工作时,会把正在使用的程序和数据从硬盘"搬"到内存里,就像把要用的书从书柜搬到课桌上，然后在课桌上使用书籍。这样处理速度就快了。等不用了,重要的东西再"收"回硬盘保存,就像把书放回书柜，其他一些不需要的东西可以直接丢到垃圾桶。

这也解释了为什么电脑突然断电,正在编辑的文档会丢失——因为那些内容还在"桌面"(内存)上,还没来得及"放回书柜"(保存到硬盘)。

**问题**：根据文章内容,下列哪项表述是**错误的**？

A. 内存的读取速度比硬盘快,但断电后数据会丢失。
B. 硬盘的存储容量通常比内存大,且断电后数据不会丢失。
C. 计算机启动程序时,程序可以直接从内存中加载并执行。
D. 在玩电脑游戏的过程中，到了存档点可以保存游戏进度，实际上是将内存数据存到了磁盘中。

## 三、数学思维能力（30分）

### 第7题（10分）

**学习新知识：二进制与异或运算**

**一、十进制转二进制**

将十进制数不断除以2，记录余数，从下往上读：

```
例1：13 转二进制
13 ÷ 2 = 6 余 1  ↑
 6 ÷ 2 = 3 余 0  |
 3 ÷ 2 = 1 余 1  |
 1 ÷ 2 = 0 余 1  |
结果：13 = 1101(二进制)
```

**二、二进制转十进制**

从右往左，第1位乘以2⁰，第2位乘以2¹，第3位乘以2²，第4位乘以2³，依次类推 ：

```
例2：1101(二进制) 转十进制
1×2³ + 1×2² + 0×2¹ + 1×2⁰
= 8 + 4 + 0 + 1
= 13
```

**三、异或运算（⊕）**

对二进制数的每一位进行运算：
- 相同得0：0⊕0=0，1⊕1=0
- 不同得1：0⊕1=1，1⊕0=1

```
例3：5 ⊕ 3 = ?
5 = 101(二进制)
3 = 011(二进制)
-----------
    110(二进制) = 6
```

**问题**：计算 13 ⊕ 7 的结果是多少？（十进制答案）

A. 6
B. 10
C. 12
D. 20

### 第8题（10分）

**学习新知识：数字根与循环数列**

**一、数字根（Digital Root）**

数字根是将一个数的各位数字相加，如果结果大于9，则继续相加，直到得到一位数。

```
例1：计算 86 的数字根
86 → 8 + 6 = 14 → 1 + 4 = 5
数字根是 5

例2：计算 789 的数字根
789 → 7 + 8 + 9 = 24 → 2 + 4 = 6
数字根是 6
```

**二、数字根循环数列**

给定一个起始数，按以下规则生成数列：
- 第1个数：起始数
- 之后每个数 = 前一个数 × 2 的数字根

```
例3：起始数为 3 的数列
第1个：3
第2个：3 × 2 = 6 → 数字根 = 6
第3个：6 × 2 = 12 → 数字根 = 1 + 2 = 3
第4个：3 × 2 = 6（开始循环）
...
数列：3, 6, 3, 6, 3, 6, ...
```

**问题**：如果起始数为 7，按照这个规则，第2025个数是多少？

## 四、综合运用能力（20分）

### 第10题（20分）

**学习新知识：CPU如何读取数据与缓存命中**

---

**一、数据从内存到CPU的过程**

计算机运行程序时，数据存放在**内存**中。但CPU（中央处理器）的运算速度非常快，而内存的读取速度相对较慢。如果CPU每次都直接从内存读数据，就会浪费大量时间等待。

为了解决这个问题，计算机在CPU和内存之间设置了**缓存（Cache）**。缓存容量小但速度极快，就像你书桌上的笔筒——虽然只能放几支笔，但拿起来比去抽屉里翻找快得多。

**数据读取流程**：
```
CPU需要数据 → 先查缓存 → 缓存有？
                         ↓是：直接使用
                         ↓否：从内存读取一大块数据到缓存，同时CPU取走想要的数据
```

---

**二、关键概念**

1. **缓存命中**：CPU需要的数据刚好在缓存中，直接使用，速度很快。
2. **缓存未命中**：CPU需要的数据不在缓存中，需要从内存一次性读取一整块内容到缓存中，同时CPU取走数据，速度较慢。
3. **缓存行（Cache Line）**：每次从内存读取数据时，不是只读1个字节，而是读取连续的一整块数据（比如64字节），这一块叫做"缓存行"

**为什么要读一整块？** 因为程序访问数据时，往往会连续访问相邻的数据（比如遍历数组）。一次多读一些，后面的访问就能直接命中缓存了。

---

**三、缓存命中率计算**

**命中率 = 缓存命中次数 ÷ 总访问次数 × 100%**


例题：假设缓存行大小为16字节，数组 A 一共有4个元素 A[0], A[1], A[2], A[3]，每个元素占4字节，它们在内存中连续存放：
```
【内存布局】（每个格子 = 4字节）

┌────────┬────────┬────────┬────────┐
│  A[0]  │  A[1]  │  A[2]  │  A[3]  │
└────────┴────────┴────────┴────────┘
|<------------- 16字节 ------------->|
```
可以看到：4个元素 × 4字节 = 16字节，刚好等于1个缓存行的大小！

CPU按顺序访问 A[0] → A[1] → A[2] → A[3]：

访问 A[0]：查询缓存，缓存为空，未命中！此时从内存读取1个缓存行（16字节）到缓存
```
┌────────────────────────────────────┐
│ 缓存内容： A[0]  A[1]  A[2]  A[3]   │
└────────────────────────────────────┘
|<------------- 16字节 ------------->|
```
访问 A[1]：查询缓存，已在缓存中，命中！
访问 A[2]：查询缓存，已在缓存中，命中！
访问 A[3]：查询缓存，已在缓存中，命中！

总结：4次访问，3次命中，1次未命中
命中率 = 3 ÷ 4 × 100% = 75%
